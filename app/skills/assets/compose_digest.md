# Compose Digest

purpose: составлять аккуратный дайджест по полученным данным (например, из email/telegram/web_search).
when_to_use: когда пользователь просит "составь дайджест", "сделай дайджест", "дайджест из писем/сообщений/результатов".
when_not_to_use: когда пользователь просит исходные данные целиком без суммаризации.
instructions:
- финальный ответ всегда на русском языке;
- если дайджест нужно составить по email, используй `read_email` и придерживайся стратегии:
  - шаг 1: `unreadOnly=true`, `markAsRead=true`, `sinceHours=24`, `maxItems=N` (обычно N=10);
  - если вернулось меньше N — сделай ЕЩЁ ОДНУ попытку: `unreadOnly=false`, `markAsRead=false`, `sinceHours=168`, `maxItems=N`;
- сделай список пунктов, разделяя пункты так: пустая строка, затем `---` (одна строка), затем пустая строка;
- каждый пункт должен содержать:
  - короткий заголовок (1 строка);
  - источник (например: email from+subject+date, либо ссылка, если есть);
  - краткое резюме (1-2 предложения);
  - короткую цитату (1-2 строки) из исходного текста/snippet (без выдумок);
- цитату делай читабельной: не включай CSS/шаблонные блоки/трекер-URL; если текст похож на технический мусор — вместо цитаты укажи "текста недостаточно для цитаты/перевода";
- если исходный текст пункта не на русском языке — добавь перевод на русский (после цитаты) и пометь, что это перевод;
- если snippet/текста не хватает — честно укажи "текста недостаточно для цитаты/перевода" и всё равно дай минимальное резюме по доступным полям (subject/from/date).
allowed_tools:
- read_email
- digest_telegram_news
- save_digest_preference
- web_search
limitations:
- не раскрывай секреты/токены/пароли;
- не выдумывай цитаты и факты; цитаты только из данных tool output.
examples:
- "Прочитай последние 5 писем и составь дайджест"

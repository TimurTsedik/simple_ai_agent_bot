# Read and Analyze Email

purpose: прочитать письма через IMAP и сделать анализ/резюме по содержимому.
when_to_use: когда пользователь просит "прочитай письма", "проверь почту", "проанализируй письма", "что нового в почте", "сводка по письмам".
when_not_to_use: когда пользователь просит работать НЕ с email или запрещает использовать инструменты.
instructions:
- финальный ответ всегда на русском языке;
- для периодических запусков (scheduler/internal run) не считай задачу "уже выполненной ранее": каждый новый run должен заново читать актуальные письма;
- используй `read_email` и придерживайся стратегии:
  - шаг 1: `unreadOnly=true`, `sinceHours=24`, `maxItems=N` (обычно N=10);
  - если вернулось меньше N — сделай ЕЩЁ ОДНУ попытку: `unreadOnly=false`, `sinceHours=168`, `maxItems=N`;
- ОБЯЗАТЕЛЬНЫЙ формат итогового ответа: ровно 3 категории в указанном порядке (даже если какая-то категория пустая):
  1. `🚨Требуют ответа/действия или предпочтительные отправители`
  2. `☝️Важные`
  3. `🗑️Остальное/мусор`
- первая строка ответа: `📨Непрочитанные письма: N✉️` (N = число писем в финальном наборе);
- далее три секции в порядке выше; каждая секция начинается со строки заголовка `### <название>` и числа писем `(M)` рядом с заголовком;
- если в секции нет писем — выведи строку `нет писем` под заголовком (но саму секцию не пропускай);
- критерии распределения по категориям:
  - категория 1 (требуют действия / предпочтительные):
    - письмо от отправителя из блока `## Email preference hints` Memory block (по точному email или подстроке домена);
    - письмо явно требует ответа/действия от получателя (вопрос, просьба, deadline, верификация, оплата, доставка, корпоративные действия по бумагам, гос/банковские уведомления, вход в аккаунт, безопасность);
  - категория 2 (важные):
    - содержательные письма без явного действия: профильные дайджесты, аналитика, новости компаний, работа/вакансии, финансы/инвестиции, технические апдейты, личная корреспонденция;
    - НЕ включать сюда промо/маркетинг (см. ниже);
  - категория 3 (остальное/мусор):
    - промо/реклама/sales/deals/coupon/распродажи/скидки/marketing/newsletter-подобные письма от магазинов;
    - типичные маркеры (в `from`/`subject`): aliexpress, ozon-promo, sale, deals, promo, shop today, save, off, скидк, распродаж, акци, "best price", "limited offer";
    - автоматические уведомления соцсетей с низкой ценностью (например LinkedIn "X profile views"), Reddit/digest-нотификации без личного запроса;
- внутри каждой секции выводи письма блоком (каждое поле с новой строки) и разделяй письма так: пустая строка, затем строка `---`, затем пустая строка;
- внутри блока показывай: `Тема`, `От`, `Дата`, `Суть` (1 предложение);
- если `snippet` содержит достаточно текста — добавь короткую цитату (1–2 строки) без выдумок;
- если `langHint=non_ru` — добавь перевод цитаты/вывода на русский и пометь как `Перевод`;
- если текста недостаточно — честно укажи `текста недостаточно для цитаты/перевода` и опирайся на subject/from/date;
- не раскрывай секреты/токены/пароли;
- не выдумывай факты и цитаты; используй только данные из tool output.
allowed_tools:
- read_email
limitations:
- максимум 2 вызова `read_email` в одном run (по стратегии выше).
examples:
- "Проверь почту за сутки и кратко расскажи, что важного"
- "Прочитай все непрочитанные письма и проанализируй"

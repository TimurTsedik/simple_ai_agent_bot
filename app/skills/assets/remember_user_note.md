# Remember user note

purpose: сохранить в долгосрочную память общий факт о пользователе по просьбе «запомни» / «запиши», сформулировав запись через LLM и вызвав `save_user_memory_note`.
when_to_use: пользователь явно просит запомнить или записать информацию о себе, своих планах, предпочтениях вне контекста дайджеста Telegram или почты (например место работы, часовой пояс, личные договорённости).
when_not_to_use:
- предпочтения по новостному дайджесту, каналам, темам — skill `telegram_digest_feedback` и tool `save_digest_preference`;
- предпочтения по письмам и отправителям — `email_preference_feedback` и `save_email_preference`;
- если пользователь говорит о понравившейся новости/статье из дайджеста и хочет «запомнить стиль» — не этот skill.
instructions:
- финальный ответ пользователю на русском;
- сформулируй `noteText`: одна короткая нейтральная фраза от третьего лица или безлично (что хранить), без лишних цитат;
- вызови `save_user_memory_note` с полем `noteText`; не выдумывай другие поля;
- после успешного tool подтверди кратко, что запомнил;
- если из сообщения неясно, что сохранять — задай один уточняющий вопрос, не вызывай tool до ответа.
allowed_tools:
- save_user_memory_note
limitations:
- не дублируй дословно длинный текст пользователя — сожми смысл;
- не используй этот tool для настроек дайджеста/почты.
json_examples:
  {
    "noteText": "Пользователь работает в сфере финансового анализа."
  }
examples:
- «Запомни, что я в отпуске до 15 июня»
- «Запиши: предпочитаю краткие ответы»

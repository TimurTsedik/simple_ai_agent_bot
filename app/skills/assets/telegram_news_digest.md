# Telegram News Digest

purpose: собирать краткий дайджест новостей из Telegram-каналов.
when_to_use: когда пользователь просит новости/дайджест/обзор.
when_not_to_use: когда запрос не связан с новостями.
instructions:
- используй tool `digest_telegram_news` для получения данных;
- извлеки из текста пользователя публичные username каналов (`@name` или `t.me/name`) и передай их в args `channels`; если пользователь не указал каналы — оставь `channels` пустым (будут каналы из конфигурации);
- если пользователь просит тематику (ИИ, экономика, крипто и т.д.), добавь соответствующие id в args `topics` (например `ai`, `economy`, `crypto`, `markets`, `tech`, `custom`); `custom` не добавляет seed-слова;
- явные слова/фразы для фильтрации передавай в `keywords`; порядок приоритета ключей: пользовательские keywords, затем seeds по topics, затем дефолты из конфигурации;
- если в Memory block есть раздел "Digest preference hints", мягко учитывай его в `topics`/`channels`/`keywords`, но явные требования пользователя в текущем сообщении важнее;
- если `digest_telegram_news` вернул `count=0`, но в `data_preview.filteredOutByTime > 0`, сделай еще один вызов с более широким окном (`sinceHours`: сначала 72, затем 168), после чего переходи к final;
- при успехе возвращай короткий итог в final;
- форматируй итог максимально читабельно:
  - первая строка: "Новости: N" (без технических префиксов);
  - далее N блоков, каждый блок разделяй: пустая строка, затем `---`, затем пустая строка;
  - блок должен быть коротким и состоять из строк:
    - `Заголовок: ...` (1 строка, убери эмодзи/хештеги в начале, не тащи служебные обрывки);
    - `Источник: <channel>, <дата>` (НЕ показывай Unix timestamp);
    - `Суть: ...` (1 короткое предложение);
    - `Цитата: ...` (1-2 строки только если текст читаемый; иначе "текста недостаточно для цитаты");
    - `Ссылка: ...` (обязательно);
- для каждой новости ОБЯЗАТЕЛЬНО добавляй прямую ссылку на конкретный пост из поля `link`;
- не заменяй ссылку текстом "канал X, сообщение #Y", если `link` присутствует;
- если данных нет, сообщай об этом явно.
allowed_tools:
- digest_telegram_news
limitations:
- не запрашивать нерелевантные источники;
- не выводить слишком длинный ответ.
examples:
- "Сделай дайджест экономичских новостей за сегодня".

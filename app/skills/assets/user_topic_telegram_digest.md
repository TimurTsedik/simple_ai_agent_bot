# User topic Telegram digest

purpose: дайджест новостей из Telegram по пользовательской теме с сохранением каналов и ключевых слов в долгосрочной памяти и учётом «непрочитанных» постов.
when_to_use: когда пользователь просит дайджест/новости **по теме** (например: «создай дайджест новостей по теме ИИ»), или хочет удалить сохранённые настройки темы.
when_not_to_use: когда нужен только общий дайджест без явной темы и без шага настройки — тогда может быть уместен skill `telegram_news_digest`.
instructions:
- финальный ответ пользователю на русском языке;
- сценарий по шагам:
  1) вызови `user_topic_telegram_digest` с `topic` (из текста пользователя), `fetchUnread=false`, `channels=[]`, `keywords=[]`;
  2) если `status=needs_channels` — спроси **только** список публичных username каналов (`@name` или `t.me/name`), затем вызови tool снова с тем же `topic` и заполненным `channels`;
  3) если `status=needs_keywords` — спроси **только** список ключевых слов/фраз для фильтрации, затем вызови tool снова с тем же `topic`, сохранёнными `channels` из tool output и новым `keywords`;
  4) если `status=ready` — вызови `user_topic_telegram_digest` с тем же `topic` и `fetchUnread=true` для загрузки данных;
  5) по успешному `fetchUnread` (`status=fetched`) собери компактный дайджест из `items` (цитаты только из полей tool);
- если пользователь попросил **удалить** настройки темы дайджеста — вызови tool с `deleteTopic=true` и тем же `topic`;
- при пустом `items` после fetch честно сообщи, что новых подходящих постов не найдено (можно кратко упомянуть `channelErrors` если есть);
- форматирование итога (как в `telegram_news_digest`): пустая строка, `---`, пустая строка между пунктами; для каждого пункта:
  - `🗞️Заголовок: ...📰` (одна строка из summary);
  - `Источник: <channel>, <дата>` (дату показывай человекочитаемо из `dateUnixTs`, без raw unix в заголовке);
  - `Суть: ...` (1 короткое предложение);
  - `Цитата: ...` (1–2 строки из summary если достаточно текста);
  - `Ссылка: ...` из поля `link`;
- не выдумывай каналы и ключевые слова: бери только из диалога или из полей `savedConfig` в ответе tool.
allowed_tools:
- user_topic_telegram_digest
limitations:
- не сохраняй секреты и персональные данные третьих лиц;
- не смешивай этот сценарий с email-дайджестом (`read_email`).
examples:
- «Сделай дайджест новостей по теме крипто» -> tool check -> уточнение каналов -> keywords -> fetchUnread -> final.

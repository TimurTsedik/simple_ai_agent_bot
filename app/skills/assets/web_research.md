# Web Research

purpose: искать информацию в интернете и давать ответ со ссылками на источники.
when_to_use: когда пользователь просит найти информацию "в интернете", "в сети", "источники", "ссылки", "что пишут", "проверь по источникам".
when_not_to_use: когда запрос можно ответить без поиска или когда пользователь просит использовать только локальные данные.
instructions:
- если требуется поиск, используй tool `web_search`;
- в одном run вызывай `web_search` не более одного раза;
- используй `fetchTopN=5` и `maxCharsPerPage=8000`, если пользователь не просит иначе;
- на основе полученных `results` и `fetchedPages` дай краткий связный ответ;
- в финальном ответе ОБЯЗАТЕЛЬНО приложи список источников ссылками (URL), минимум 2 источника если они есть;
- не придумывай факты, которых нет в полученном тексте страниц; если источники противоречат — укажи это.
- если в памяти/истории есть прошлые ошибки `ACCESS_DENIED` или похожие — всё равно выполни ОДНУ попытку `web_search` в этом run (не повторяй инструмент после этого).
allowed_tools:
- web_search
limitations:
- не пытайся обходить ограничения URL (localhost/private IP запрещены);
- не делай много запросов — один tool-call за run;
- если выдача пустая — честно сообщи об этом.
examples:
- "Найди в интернете, что означает ошибка X в FastAPI"
